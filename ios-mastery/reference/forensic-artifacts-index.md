# Forensic artifacts index

This is a **derived reference**, mechanically compiled from the artifact annotations in every lesson across all twelve parts of the iOS curriculum and then deduplicated: an artifact named in several lessons becomes **one row** whose `Covered in` column unions every lesson that mentions it. The canonical deep treatment of any single store lives in its lesson — start at [[01-knowledgec-db-deep-dive]], [[02-biome-and-segb-streams]], [[04-communications-imessage-and-sms]], [[07-location-history]], or [[03-the-itunes-finder-backup-format]] depending on the store.

> [!warning] Authorized use only
> Every item below is **read-only evidence**. Establish authority before access, **image-then-examine**, **copy-before-query** (a bare `SELECT` write-locks SQLite and spawns `-wal`/`-shm`), capture a tool-version manifest, and **never reboot a seized device** — a reboot drops AFU→BFU and evicts the resident Class-C keys. See [[00-ios-forensics-landscape-and-authorization]] and [[00-how-to-use-this-course]].

## How to read this index

**Encryption marker.** On iOS *everything on the Data volume is Data-Protection-encrypted at rest* (per-file keys; the default is Class C / `NSFileProtectionCompleteUntilFirstUserAuthentication`, recoverable AFU). So a generic "it's encrypted" flag would be noise. **🔒 therefore marks only an *additional* cryptographic constraint** beyond baseline Class-C: the Keychain and keybags, SEP-bound items, Class A/B files (which go dark on lock even while AFU), FairPlay key material, app-level SQLCipher/SEE encryption (key in Keychain), encrypted-backup-only stores, and host-side pairing/escrow key material. Details: [[02-data-protection-and-keybags]], [[03-passcode-bfu-afu-and-inactivity]], [[08-keychain-on-ios]].

**Acquisition tier** (the column) — the *least* invasive method that yields the artifact:

| Tier | Meaning |
|---|---|
| **FFS** | Full-file-system — the Data volume in the clear (jailbreak / checkm8 / `usbliter8` / agent / commercial physical). The bulk of pattern-of-life. |
| **Backup** | iTunes/Finder `mobilebackup2` logical backup (no jailbreak). *Encrypted* backup adds Health + a decryptable Keychain. |
| **AFC** | `com.apple.afc` media partition + `house_arrest` per-app `Documents/` (no jailbreak, AFU). |
| **Adv-logical** | AFU logical aided by a host **pairing/escrow record** (no passcode re-entry). |
| **Device-services** | `lockdownd`-brokered services over usbmux: `sysdiagnose`, crash copy, `idevicesyslog`, diagnostics relay. |
| **Cloud** | iCloud / CloudKit server-side (legal process or token replay; ADP closes most of it). |
| **Firmware** | Public IPSW research substrate — *not* a subject-device acquisition (kernelcache, dyld cache, bootloaders, SEP fw). |
| **Host-side** | Lives on the **paired Mac/PC**, not the device (lockdown records, paired-Mac `chat.db`, iPhone-Mirroring caches, Find My on macOS). |
| **Simulator** | Mac CoreSimulator — schema/layout only, **no SEP/DP/AMFI** (teaches structure, never real evidence). |
| **SIM** | Physical SIM/UICC read over a PC/SC reader (EF files; not extractable from an eSIM). |
| **Below-FS** | NAND-level / Effaceable Storage; chip-off yields only AES ciphertext. |

Column order matches the lesson annotations: `| Artifact / path | Format | What it reveals | Acquisition tier | Covered in |`.
