# Keyboard-Shortcut Master Sheet

> **In one sentence:** Every macOS shortcut you'll reach for daily — system-wide, Finder, text navigation, window management, browser, screenshots, boot sequences, and beyond — organized for instant lookup and print-ready reference.

> 🔬 **Forensics note:** `⇧⌘.` (toggle hidden files) is the fastest way to expose `.DS_Store`, `._*` AppleDouble files, and `.Spotlight-V100` index directories to visual inspection. No Terminal needed. The state persists in `com.apple.finder ShowAllFiles` via `defaults`.

> 🔬 **Forensics note:** `⌘I` on a file reveals POSIX timestamps (Created, Modified, Last Opened), the file's quarantine state ("Where From"), and extended attributes hint. For deeper inspection use `xattr -l <file>` and `mdls <file>` in Terminal — see [[part-01-architecture/09-spotlight-metadata-and-xattrs]] and [[part-05-security-forensics/03-forensic-artifacts]].

**Move files (the Windows "Cut" equivalent):** Finder has no `⌘X` for files. The workflow: `⌘C` to copy, navigate to destination, then `⌘⌥V` to move (not `⌘V`, which copies). The destination must be on the same volume for a true move; cross-volume always copies then trashes the original.

## 3 — Text Editing & Navigation

These bindings are **built into the AppKit text system** and work in every native text field: Messages, Notes, Safari address bar, Terminal, Mail, system dialogs — everywhere. They are Emacs-derived and date to NeXTSTEP. Knowing them eliminates 80% of your need for the arrow keys.

> 🪟 **Windows contrast:** Windows text navigation uses `Home`/`End`/`Ctrl+←/→`. **Custom text bindings:** The file `~/Library/KeyBindings/DefaultKeyBinding.dict` lets you add arbitrary Emacs-style bindings in a plist format. Changes take effect after relaunch. See [[part-02-gui/04-keyboard-shortcuts-and-customization]] for the full authoring guide.

## 10 — Boot & Startup Key Combinations

Boot keys must be held during power-on, **immediately after the startup chime or the moment the screen lights up**, and released when the relevant screen appears.

### Apple Silicon (M-series, 2020+)

On Apple Silicon, most boot options appear in a graphical **Startup Options** screen rather than requiring held keys during boot.

| How to invoke | Action |
|--------------|--------|
| **Hold power button** until "Loading startup options…" appears, then release | Enter Startup Options (Recovery, Startup Disk, Safe Mode selector) |
| In Startup Options, select volume, then hold `⇧` and click "Continue in Safe Mode" | **Safe Mode** — loads only essential kexts, clears font caches, rebuilds Spotlight index |
| Choose "Options" in Startup Options, then log in | **Recovery Mode** (macOS Recovery — Disk Utility, Terminal, Reinstall) |
| In Recovery Terminal: `csrutil disable` | Disable SIP (requires Reduced Security in Startup Security Utility) |

> ⚠️ **ADVANCED / DESTRUCTIVE — Apple Silicon Recovery:** Reducing security on Apple Silicon (Permissive Security or Reduced Security) requires your Apple ID and administrator password. `csrutil disable` alone on AS also disables the Sealed System Volume signature. Document the current state first: `csrutil status; bputil -d`. Roll back: re-enter Recovery, choose Startup Security Utility, restore Full Security.

### Intel / T2 (pre-2020, and 2020 Intel models)

On Intel Macs, keys are held at startup in the traditional BIOS-style manner:

| Keys to hold | Action |
|-------------|--------|
| `⌘ R` | macOS Recovery (latest installed macOS) |
| `⌘ ⌥ R` | Internet Recovery (latest macOS compatible with hardware) |
| `⌘ ⇧ ⌥ R` | Internet Recovery (macOS that came with the Mac, or closest available) |
| `⌥` (Option) | Startup Manager — choose boot volume (shows all volumes + network) |
| `⌃` (hold during Startup Manager selection) | Set selected volume as default startup disk |
| `⇧` | Safe Mode (loads only essential extensions; hold until Apple logo with progress bar) |
| `⌘ V` | Verbose mode (kernel messages in text; must disable SIP or use `nvram boot-args="-v"`) |
| `⌘ S` | Single-user mode (shell, pre-mount — removed in macOS Monterey 12+) |
| `T` | Target Disk Mode (exposes Mac as a Thunderbolt/FireWire disk to another Mac) |
| `D` | Apple Diagnostics (hardware test) |
| `⌥ D` | Apple Diagnostics via Internet |
| `N` | NetBoot / Network Startup (boot from server) |
| `⌘ ⌥ P R` | Reset NVRAM/PRAM (hold through second startup chime; repeat twice) |
| `⌃ ⌘ Power` | Force restart (hardware — not a soft reboot) |

> 🔬 **Forensics note:** Target Disk Mode (`T` at boot, Intel only) turns the Mac into a Thunderbolt/USB storage device, allowing bit-for-bit acquisition with `dd` or `dc3dd` from a second machine. Apple Silicon replaced this with **Share Disk** mode, activated from Recovery Mode (Actions menu ▸ Share Disk) rather than a boot key. The underlying disk still appears as a mass-storage device to the examiner Mac.

> 🔬 **Forensics note:** NVRAM reset (`⌘⌥PR`, Intel) clears `boot-args`, display resolution, startup disk override, and some security flags stored in NVRAM — potentially destroying evidence of deliberate boot-arg manipulation (e.g., `-s` single-user flags left from a prior intrusion). Image NVRAM contents first with `nvram -x -p > nvram-backup.plist` before any reset.

> ⚠️ **ADVANCED — T2 Macs:** On 2017–2020 Intel Macs with the T2 security chip, some boot-key operations (especially SIP disable, Recovery mode for OS reinstall) also require the Startup Security Utility. T2 SMC reset: hold `⌃ ⌥ ⇧` on the **right** side of the keyboard + power for 7 seconds, release, then press power again.

See [[part-01-architecture/01-boot-process]] and [[part-01-architecture/02-apple-silicon-soc-and-secure-enclave]] for the full boot-architecture lesson.

## 12 — Discovering & Customizing Shortcuts

### Discovering What Exists

1. **Menu bar first.** Every app command with a shortcut shows the symbol sequence to the right of the menu item — this is the definitive source. Open a menu and read it.
2. **Help menu search.** `⌘?` opens the Help menu and accepts text search — it highlights the matching menu item in place, even across submenus.
3. **`System Settings ▸ Keyboard ▸ Keyboard Shortcuts`** lists all system shortcuts and lets you add per-app menu overrides.
4. **`defaults read com.apple.symbolichotkeys AppleSymbolicHotKeys`** dumps all current system shortcut assignments as a plist. IDs and mappings are documented in community references (Brett Terpstra's list is canonical).

### Customizing via System Settings

**System Settings ▸ Keyboard ▸ Keyboard Shortcuts:**

- **App Shortcuts** (bottom of the list): Add any entry with `Application`, the exact menu item text (copy it verbatim — capitalization matters, as the match is string-exact against the menu hierarchy), and the desired key combo. This rebinds the shortcut at the NSMenu level for that app.
- **Spotlight, Screenshots, Mission Control, Services** panels each allow changing their respective hardwired shortcuts.

**System-wide app shortcut example** — rebind "Close Tab" from `⌘W` to something that doesn't accidentally close terminal sessions:

```
Application: Terminal
Menu Title:  Close Tab
Keyboard Shortcut: ⌥⌘W
```

### Customizing at the Kernel Level (hidutil)

`hidutil property --set` remaps keys at the HID (Human Interface Device) layer — below any application, before the key event reaches the OS. Effective for swapping `Caps Lock → Escape` for vim users, or remapping keys on non-standard keyboards.

```bash
# Remap Caps Lock (0x700000039) to Escape (0x700000029)
hidutil property --set '{"UserKeyMapping":[{"HIDKeyboardModifierMappingSrc":0x700000039,"HIDKeyboardModifierMappingDst":0x700000029}]}'
```

Changes are lost on reboot; persist via a `launchd` plist at `~/Library/LaunchAgents/`. See [[part-02-gui/04-keyboard-shortcuts-and-customization]] for the full `hidutil` authoring pattern.

## 13 — Globe / fn Key (Modern Keyboards)

The Globe key (`🌐`) on Apple Silicon keyboards (MacBook Pro 2021+, Magic Keyboard with Touch ID, MacBook Air M2+) is a new modifier distinct from `fn`:

| Shortcut | Action |
|----------|--------|
| `Globe` (tap) | Dictation (single tap) |
| `Globe Globe` | Toggle emoji / character viewer |
| `Globe E` | Emoji & Symbols |
| `Globe F` | Full-screen toggle |
| `Globe H` | Return to Desktop (like `⌘M` all; varies) |
| `Globe M` | Zoom (open Launchpad on some configs) |
| `Globe Q` | Quick Note (macOS Monterey+) |
| `Globe ↑` / `Globe ↓` | Mission Control / App Exposé |
| `Globe ←` / `Globe →` | Move to Space left/right (= `⌃←` / `⌃→`) |
| `Globe fn 1–9` | Function keys F1–F9 when configured as standard |

Configure Globe key behavior: **System Settings ▸ Keyboard ▸ Press Globe/fn key to…** (options: Do Nothing, Change Input Source, Show Emoji & Symbols, Start Dictation, Show/hide macOS Sequoia features).

## 14 — Universal Keyboard Conventions by Semantic Role

Understanding the *why* behind modifier usage lets you predict unknown shortcuts:

| Want to… | Default pattern | Examples |
|-----------|----------------|---------|
| Standard app command | `⌘ key` | `⌘S` save, `⌘P` print, `⌘Q` quit |
| Inverse / undo of that command | `⇧⌘ key` | `⇧⌘Z` redo, `⇧⌘T` reopen tab |
| Variant / extended version | `⌥⌘ key` | `⌥⌘V` paste & match style, `⌥⌘W` close all |
| More powerful / system-level | `⌃⌘ key` | `⌃⌘Q` lock, `⌃⌘F` full-screen, `⌃⌘Space` emoji |
| Secret / hidden option | Hold `⌥` over any menu | Many menu items reveal alternate actions |
| Repeat effect on multiple items | Hold `⌥` while clicking some checkboxes | Column-collapse in List view, etc. |
| Move cursor at text level | `⌃ letter` | All Emacs bindings (`⌃A/E/K/F/B/N/P`) |
| Extend selection | `⇧ + any movement` | `⇧⌥→` extend word, `⇧⌘↓` extend to doc end |
