# Handoff — how to continue building this curriculum

This file is the canonical instruction set for **extending or revising** the
macOS Mastery course. Read it before adding or rewriting lessons.

## What this is

A self-paced macOS power-user curriculum for **one specific learner**: a
computer-forensics professional and software builder who switched from Windows
and wants top-1% mastery, *with engineering depth*. Target platform is
**macOS 26.x (Tahoe-era) on Apple Silicon**, Intel/T2 history noted where
relevant. It lives in `~/dev/PhantomLives/macos-mastery/` and is mirrored into
Obsidian by the repo's `sync-md-to-obsidian.sh` (git-tracked `.md` only — so
**commit new lessons** or they won't appear in Obsidian).

## File/naming conventions

- Lessons: `part-NN-theme/NN-slug.md`. Two-digit, zero-padded numbers.
- References: `reference/<name>.md`.
- The authoritative lesson list + build status is [CURRICULUM.md](CURRICULUM.md). **Update its status column** (⬜→🚧→✅) whenever you touch a lesson.
- Cross-link with Obsidian `[[wikilinks]]` *and* relative Markdown links where a clickable repo link helps. Prefer relative Markdown links for navigation tables; use `[[slug]]` inline when referencing a concept covered elsewhere.

## Voice & depth rules

- **Assume competence.** No "click the Apple menu" hand-holding unless the *point* is a non-obvious GUI path. The learner is technical.
- **Explain the mechanism.** Always answer "how does this actually work?" — name the daemon, the file, the on-disk format, the syscall, the framework.
- **Labs are full-power.** Destructive/admin operations are allowed inline, but every one gets a ⚠️ warning with how to back up first and how to roll back.
- **Currency matters.** Verify version-specific facts against current macOS (26.x). Flag where behavior changed (e.g. `cron`→`launchd`, kexts→System Extensions, `ifconfig` deprecation nuance, SMC/NVRAM resets being Intel-only).
- **Cite real sources** in Further reading: `man` pages, Apple Platform Security guide, Apple Developer docs, Howard Oakley's *Eclectic Light Company*, Reddit r/macOS & r/MacOSBeta, GitHub tool repos, Jamf/Mac-admins community.

## Reference spines — keep them in sync

When a lesson introduces a term/acronym/shortcut/command, make sure it also
lands in the matching reference file (`glossary.md`, `acronyms.md`,
`keyboard-shortcuts.md`, `cli-cheatsheet.md`). The references are the
consult-constantly layer; lessons are the learn-once layer.

## Build status

The initial corpus was generated by a multi-agent research workflow (see the
session that created this folder). When continuing:

1. Pick the next 🚧/⬜ lessons from CURRICULUM.md.
2. Research current behavior (don't write from stale memory for version-specific claims).
3. Write to the template. Update CURRICULUM status.
4. Add new terms to the reference spines.
5. `git add` the new files + commit + push, then `./sync-md-to-obsidian.sh` so Obsidian updates.
