# Moving Jan's calendars to the hosted version (screen-share checklist)

The hosted app lives at a **different web address** than the old file copied to her
laptop, so calendars saved in the old version **do not appear automatically** in
the new one. They have to be exported from the old app and imported into the new
one. Don't ask Jan to do this alone — do it together on a screen-share (or remote
control her screen). Plan ~10–15 minutes.

Jan is on **Windows 11 with an external mouse**, low vision — go slowly, and use
**Ctrl** + **+** to zoom her screen so she can follow.

## Before the call

- Have the new link ready: **https://bronty13.github.io/calendarmaker/**
- Ask Jan roughly **how many calendars** she has in the old version.
- If she has **none worth keeping** → skip everything below; just have her open the
  new link and bookmark it (star ☆ → Done), and start fresh.

## Step 1 — Open BOTH versions

1. Open the **old** version (her current bookmark / the file on her laptop).
2. Open the **new** version in a second tab: https://bronty13.github.io/calendarmaker/
3. **Leave the old one alone for now** — don't delete or change anything until every
   calendar is confirmed in the new app.

## Step 2 — Export each calendar from the OLD app

Do this once per calendar:

1. On the old app's home screen, **click the calendar** to open it.
2. Click **Export PDF** (top of the screen).
3. In the box that opens, click **Export bundle (.cmcal.json)** (bottom-left).
4. A small file downloads — named after the calendar, e.g.
   `family-june-2026.cmcal.json` — into the **Downloads** folder.
5. Close the box, click **← All calendars**, and repeat for the next one.

Tip: press **Ctrl** + **J** in the browser to see the list of downloaded files and
confirm each one saved.

## Step 3 — Import them into the NEW app

1. Switch to the **new** tab (the hosted link).
2. On the home screen, click **Import…**.
3. A file window opens → go to **Downloads** → **double-click** the first
   `.cmcal.json` file.
4. The calendar appears and opens. Click **← All calendars**.
5. Click **Import…** again for the next file, until all are in.

## Step 4 — Check everything came over

- Every calendar from the old app now shows under **Your calendars** in the new
  app.
- Open one or two and glance at the days to make sure they look right.

## Step 5 — Finish up

1. Make sure the **new** link is bookmarked (star ☆ → **Done**).
2. To avoid confusion, **remove the old bookmark** (or rename it "OLD — don't
   use"). The old file on the laptop can be deleted later — no rush.
3. Going forward she only uses the new bookmark.

## If something's off

- **A calendar didn't import:** re-export it from the old app (Step 2) and import
  again.
- **Can't find the file in the Import window:** look in **Downloads** and sort by
  date (newest first).
- **Anything unexpected:** stop and keep the old version — nothing is lost as long
  as the old app is untouched. Try again, or move that calendar by hand.

Nothing here can lose data: exporting and importing only **copies**. The old
version stays exactly as it is until you choose to remove it.
